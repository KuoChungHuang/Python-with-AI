# MarkItDown 桌面轉檔工具 — 開發規格書

版本：v0.2（已確認範圍，可開始開發）
日期：2026-07-07
專案目錄：`D:\Cowork Project\Markitdown`

---

## 1. 專案目標

提供一個桌面小工具，讓使用者將任意檔案（PDF、Word、Excel、PPT、圖片、音訊等）或**整個資料夾**拖拉到視窗中，
程式會呼叫 [microsoft/markitdown](https://github.com/microsoft/markitdown) 套件將檔案轉換成 `.md`（Markdown）檔案，
並將轉出的 `.md` **存放在來源檔案的同一目錄下**，不需要使用者打開命令列。

## 2. 技術選型

| 項目 | 選擇 | 說明 |
|---|---|---|
| 語言 | Python 3.12 | 沿用已安裝的環境（`C:\Users\user\AppData\Local\Programs\Python\Python312`） |
| GUI 框架 | PySide6 (Qt for Python) | 原生支援拖拉、視覺回饋佳，官方 LGPL 授權可免費商用 |
| 轉檔引擎 | `markitdown[all]` | 已安裝於系統 Python 環境；PDF 另外直接使用其相依套件 `pdfminer.six` 做純文字萃取（見 4.1） |
| 執行方式 | `.bat` 一鍵啟動腳本 | 自動檢查/建立 venv、安裝依賴、執行 `main.py`，非工程師使用者雙擊即可，不需手動下指令 |
| 多執行緒 | `QThread` | 轉檔工作放到背景執行緒，避免大檔案轉換時介面凍結；支援使用者中途取消 |

## 3. 功能需求

### 3.1 核心功能（v0.1 全部納入）
1. **拖拉輸入**：使用者可將一個或多個檔案、或**整個資料夾**從檔案總管拖拉進視窗；也提供「選擇檔案」按鈕作為替代輸入方式。拖入資料夾時程式自動**遞迴**尋找其中可轉換的檔案並加入佇列。
2. **批次轉換**：一次拖入多個檔案（或資料夾展開後的檔案）時，逐一排隊轉換，介面顯示整體與個別進度。
3. **同目錄輸出**：轉出的 `.md` 檔案存放在來源檔案所在資料夾，檔名與來源相同（副檔名改為 `.md`）。
4. **檔案總覽清單**：顯示已加入佇列的檔案、狀態（等待中／轉換中／成功／失敗／已略過）。
5. **結果開啟**：轉換成功的項目可雙擊，用系統預設程式開啟輸出的 `.md`；或提供「開啟所在資料夾」按鈕。
6. **錯誤處理與紀錄**：
   - 不支援的格式 → 清單標示「失敗」，並在下方日誌區顯示原因。
   - 目的地已存在同名 `.md` → **每次都彈出詢問視窗**（覆蓋 / 略過 / 另存為，自動加流水號），不預設任何選項，需使用者主動點擊；可勾選「套用到全部」讓使用者一次決定佇列中其餘同類衝突的處理方式。
   - 單一檔案轉換失敗不影響佇列中其他檔案繼續執行。
7. **取消與正常關閉**：轉換過程中可按「取消」中止剩餘佇列；使用者關閉視窗時，背景執行緒需安全中斷（不強制 kill、不留下寫到一半的殘缺 `.md`），已完成的檔案結果予以保留。
8. **日誌區**：視窗下方提供可捲動的文字日誌，記錄每個檔案的處理時間與結果，方便使用者追蹤或複製回報問題。

### 3.2 次要功能（v0.1 之後視需要再開發）
- 可自訂輸出目錄（而非固定同目錄）。
- 記住上次視窗大小、位置。
- 深色模式（跟隨系統主題）。

## 4. 非功能需求

- **易用性**：主要操作（拖拉→轉換→取得結果）不超過 2 個滑鼠動作。
- **穩定性**：單一檔案轉換失敗需被程式捕捉（try/except），不可讓整個應用程式崩潰；目的地資料夾無寫入權限（唯讀資料夾、受保護網路磁碟）時，該檔案標示失敗並顯示原因，不影響其他檔案。
- **相容格式**：以 `markitdown[all]` 支援的格式為準，包含 PDF、DOCX、XLSX、PPTX、圖片（含 OCR 描述）、音訊（語音轉文字）、HTML、CSV、JSON、XML、EPUB、YouTube 連結等。
- **中文檔名／路徑**：完整支援含中文、空白、括號的檔名與路徑（沿用先前手動測試已驗證可行）。

### 4.1 PDF 輸出限制（M0 POC 已驗證）

`markitdown` 內建的 PDF 轉換器會嘗試用文字座標推測表格結構、輸出 Markdown 表格語法，但遇到複雜或跨欄版面（實測政府採購文件）時常誤判，產生排版錯亂、斷行破碎的 pipe 表格，可讀性不及格。

**決定：PDF 一律只輸出純文字，不產生表格。** 實作上，`converter.py` 對 `.pdf` 不呼叫 `markitdown` 的預設分派，而是直接呼叫其相依套件的 `pdfminer.high_level.extract_text()` 取得純文字內容，完全跳過表格偵測邏輯。其餘格式（DOCX/XLSX/PPTX 等）維持呼叫 `markitdown.MarkItDown().convert()`，其原生表格輸出品質良好，不受影響。

## 8. 同名檔案衝突處理策略

拖入 `A.pdf`，同目錄若已存在 `A.md` 時，**每次都彈出對話框**供選擇（不預設任何選項，必須主動點選才能關閉對話框）：

1. **覆蓋**：直接取代既有的 `.md`。**不可逆，預設仍然逐一詢問**——只有勾選「套用到全部」才會讓覆蓋套用到佇列中其餘同名衝突的檔案。
2. **略過**：不轉換此檔案，清單標示「已略過」。風險低，選了就**自動套用**到這批其餘的同名衝突，不需要另外勾選。
3. **另存新檔**：自動命名為 `A(1).md`、`A(2).md`…風險低，選了就**自動套用**到這批其餘的同名衝突，不需要另外勾選。

（實測回饋 M4：一開始「套用到全部」設計成單一勾選框套用在任一選項上，導致每次批次轉換都要為了「另存新檔」多按一次勾選框才能免去逐檔詢問；改成依風險分流——安全的選項自動套用、有破壞性的選項維持逐一確認——才符合實際使用體驗。）

## 10. 測試計畫

- **格式覆蓋測試**：PDF、DOCX、XLSX、PPTX、PNG/JPG、MP3 各挑一個檔案測試轉換成功率。
- **PDF 純文字輸出測試**：確認 PDF 轉出的 `.md` 不含破碎的 pipe 表格語法，內容為可讀的純文字／段落。
- **中文路徑測試**：使用含中文、括號、空白的檔名與路徑（沿用資料夾內既有的 3 個 PDF 作為測試素材）。
- **批次測試**：一次拖入 5+ 個混合格式檔案，確認佇列與進度顯示正確。
- **資料夾拖拉測試**：拖入含子資料夾與多種格式檔案的資料夾，確認遞迴掃描與佇列展開正確。
- **例外測試**：拖入不支援格式（如 `.exe`）、無副檔名檔案、已被其他程式鎖定的檔案、輸出目的地為唯讀資料夾，確認錯誤訊息清楚且不影響其他檔案。
- **同名衝突測試**：驗證覆蓋／略過／另存新檔三種選項行為正確，且「套用到全部」勾選與不勾選時行為皆正確。
- **取消/關閉測試**：轉換中途按「取消」或關閉視窗，確認背景執行緒安全中止、已完成項目結果保留、不留殘缺檔案。
