# 「見樹不見林」架構定位功能 — 討論記錄

狀態：討論中，尚未進入實作

## 1. 問題背景

使用者在練習題庫時常「見樹不見林」——答題當下不知道這一題在政府採購法規與實務的整體架構中屬於哪個位置。希望在答題介面上，讓使用者一邊作答一邊了解題目在法律架構下的位置。

## 2. 已確認的方向

### 2.1 呈現方式
- 題目上方加**麵包屑**（例如「招標與投標程序 › 廠商資格設定 › 特定資格」），顯示當前題目在架構中的位置。
- 搭配可展開的**迷你架構樹**，當前節點高亮。
- 進一步構想：架構樹本身可以變成**新的練習入口**——點某個節點就練習屬於該主題的題目，讓使用者可以照法律邏輯脈絡練習，而不只是照 PDF 原本的 14 個實務分類。

### 2.2 採用的架構（使用者提供，第二版，已確認採用）

```
政府採購法規與實務作業體系
├─ 基本原則與法規架構

│  ├─ 採購類別（工程／財物／勞務）
│  └─ 採購倫理與防弊（利益衝突迴避／禁止請託關說／資訊保密義務／採購專業人員資格）
├─ 金額級距與招標決標方式
│  ├─ 法定金額門檻（巨額／查核金額／公告金額／小額）
│  ├─ 招標方式（公開／選擇性／限制性）
│  ├─ 特殊採購模式（統包／共同供應契約／電子採購）
│  └─ 決標原則（最低標／最有利標／複數決標）
├─ 招標前置與評選作業
│  ├─ 招標文件與規格（技術規格／同等品／廠商資格／押標金保證金）
│  ├─ 預算與底價（價格分析／工程價格資料庫／底價訂定核定）
│  └─ 評選機制（評選委員會／工作小組初審／評選項目權重／委員評分差異處理）
├─ 履約管理與驗收結算
│  ├─ 履約控管（履約期限展延／專案管理技師簽證／分包禁止轉包／估驗計價物價調整）

│  └─ 驗收作業（初驗複驗／部分驗收減價收受／逾期違約金／結算驗收證明書）
├─ 爭議處理與違規裁罰
│  ├─ 權益救濟（異議申訴／採購申訴審議委員會／履約爭議調解／仲裁訴訟）
│  └─ 違約處置（追繳不予發還押標金／終止解除契約／停權處分／損害賠償）
└─ 常見缺失與錯誤態樣
   ├─ 招標階段缺失（規格綁標／資格過當／文件矛盾漏列）
   ├─ 決標階段缺失（標價偏低處置不當／評選程序錯亂／決標資訊傳輸錯誤）
   └─ 履約驗收缺失（擅准停工展延／監造不實查驗造假／驗收放水超量估驗／效益不彰）
```

### 2.4 資料模型關鍵決定：一題可掛多個標籤

題目對節點的關係**不是一對一**，而是**一對多**。原因：
- 道德規範及違法處置類題目要能同時掛兩個標籤。
- 工程及技術服務採購作業、財物及勞務採購作業本身是從實務操作角度出發，天生橫跨多個架構節點，這是正常現象，**不須被單一節點歸屬所困**，分類上本來就該多重歸屬。
- 政府採購全生命週期概論則掛固定的「總覽」標籤。

資料結構草案：

```json
{
  "id": "eng-single-001",
  "framework_tags": ["履約管理與驗收結算/履約控管", "招標前置與評選作業/招標文件與規格"]
}
```

即 `framework_tags` 為陣列而非單一欄位。

## 3. 標記方法規劃（進行到一半，已暫停）

- 解析 PDF 時，已順手從第 12-14 類（政府採購法之總則招標決標／履約管理驗收／罰則附則）留下 **944 題**帶有 `source_legal_ref`（法條依據，如「第22條」），存在 [questions.json](D:\Cowork Project\政府採購法\website\src\data\questions.json)。
- 構想：先建一份「法條 → 架構節點」對照表，讓這 944 題自動精確定位，不用逐題人工判斷。
- 已統計出這 944 題實際涉及 **84 個不同法條**（含出現次數），最高頻的是第22條（154次）、第36條（48次）、第94條（35次）、第25條/第30條（各31次）等。
- 我已開始草擬 84 條的對照表，但**尚未完成、也未經法規核對**——草擬依據是訓練知識，非逐條查證現行政府採購法條文，法條可能因修法而變動，正式使用前必須人工核對。
- 剩餘約 **2,650 題**（來自其餘 11 個 PDF 分類）完全沒有 `source_legal_ref`，需要類別層級對照或 AI 輔助才能標記。

## 4. 中止討論的關鍵洞察（本次暫停原因）

使用者指出一個重要的規劃順序問題：

> 網站規劃的「進階版」目標，是讓使用者不只知道答案，還要知道**答案的理由**及**法條依據**（對應 [政府採購法練習網站_開發規格書v1.md](D:\Cowork Project\政府採購法\政府採購法練習網站_開發規格書v1.md) 第二階段「解析與法規依據」）。
>
> 但題庫裡**很多題目根本沒有法條依據**（只有 944/3,599 題有，約 26%），**更不用說目前完全沒有製作「解析」內容**。

這代表：「架構定位標記」這個功能，跟規格書 Phase 2「解析＋法規依據」的內容製作工作**高度相關，甚至可能該合併規劃**，不宜分開單獨進行，否則可能順序顛倒或重工——例如，如果先花力氣做架構標記，之後做 Phase 2 解析製作時又要重新處理法條依據的判斷，等於做兩次類似的事。
